API Reference
=============

This section describes the APIs and key functionalities provided by MindMira.

Endpoints
---------

1. **/analyze_emotion**
   - **Method**: POST
   - **Description**: Analyzes the user's input to detect emotional states.
   - **Request Body**:
     ```json
     {
       "text": "I feel stressed and anxious."
     }
     ```
   - **Response**:
     ```json
     {
       "emotion": "stressed",
       "confidence": 0.95
     }
     ```

2. **/generate_response**
   - **Method**: POST
   - **Description**: Generates a chatbot response based on user input.
   - **Request Body**:
     ```json
     {
       "text": "What should I do to feel better?"
     }
     ```
   - **Response**:
     ```json
     {
       "response": "Try taking deep breaths and focusing on positive thoughts."
     }
     ```

3. **/log_session**
   - **Method**: POST
   - **Description**: Logs the user's current session for analysis and future improvement.
   - **Request Body**:
     ```json
     {
       "user_id": "12345",
       "conversation": [
         {"text": "I feel sad.", "emotion": "sad"},
         {"text": "What can I do?", "response": "Try to talk to someone you trust."}
       ]
     }
     ```
   - **Response**:
     ```json
     {
       "status": "success",
       "message": "Session logged successfully."
     }
     ```

How to Use the APIs
--------------------

- **Authorization**: Currently, no authentication is required, but future versions may include token-based authentication.
- **Testing**: APIs can be tested using tools like `Postman` or `cURL`.
- **Error Handling**:
  - Example error response:
    ```json
    {
      "error": "Invalid input format",
      "status_code": 400
    }
    ```

Future Enhancements
-------------------

- Additional APIs for user profiling and personalized recommendations.
- Integration with third-party services for deeper emotional analysis.
- Real-time API documentation using tools like Swagger.
